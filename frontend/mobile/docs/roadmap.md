# Dreamate 商业化 Roadmap（对齐 Love Dovey）

目标：把当前「本地单机 + 客户端直连 LLM」升级为「可上架、可付费、可运维」的完整产品形态，满足 App Store / Google Play 审核要求。

## 范围与假设
- MVP 核心：AI 角色聊天 + 词库/成长 + 钱包/订阅 + 账号体系 + 云同步 + 合规与风控
- 非目标（短期不做）：复杂社交/UGC 社区、直播、完整电商
- 假设：后端可用（自建或 BaaS），LLM/TTS 走服务端代理

## 里程碑（建议节奏，可根据团队资源调整）

### M0 立项与架构基线（T+0 ~ 1 周）
- [ ] 明确商业模式：订阅 / 心动币 / 混合
- [ ] 确定后端方案（自建 / Supabase / Firebase）与数据模型
- [ ] 明确 AI 网关策略：模型选择、计费方式、限额与回退
- [ ] 密钥迁移策略：所有敏感 Key 从客户端移除
- 退出标准：架构图 + 数据模型 + 收费策略 + 关键风险清单

### M1 账号体系与云端基础（T+1 ~ 3 周）
- [ ] 正式账号注册/登录（含邮箱验证、找回密码）
- [ ] Session / Token / 设备绑定机制
- [ ] 用户画像与设置云端存储（替代本地 DB）
- [ ] 访客迁移到正式账号
- 退出标准：可在多设备登录并同步设置与历史

### M2 AI 网关与内容安全（T+2 ~ 4 周）
- [ ] LLM/TTS 全量走服务端代理
- [ ] 计费与限额：请求级计费、超额处理
- [ ] 内容安全：输入/输出审核、敏感话题拦截、举报链路
- [ ] Prompts 管理与版本化（便于热更新与回滚）
- 退出标准：客户端无密钥、服务端可控成本与安全策略

### M3 支付与权益体系（T+3 ~ 6 周）
- [ ] IAP / 订阅接入（iOS/Android）
- [ ] 服务端票据校验与权益管理
- [ ] 心动币/订阅权益结算与消耗日志
- [ ] 购买恢复与退款处理
- 退出标准：完整可用的支付闭环与防刷机制

### M4 数据同步与运营能力（T+4 ~ 7 周）
- [ ] 聊天/词库/成长数据云同步与备份
- [ ] 推送与运营触达（服务端推送）
- [ ] 运营后台：角色/内容/配置管理
- [ ] 远程配置 / A-B / 灰度发布
- 退出标准：运营可独立配置内容与活动

### M5 上架合规与发布准备（T+6 ~ 8 周）
- [ ] 隐私政策/服务条款/AI 免责声明
- [ ] 年龄分级与未成年保护
- [ ] 账号注销、数据导出/删除
- [ ] 崩溃监控、埋点与留存分析
- [ ] 上架素材与审核材料
- 退出标准：满足 App Store / Play 审核要求并可提交

- [ ] 付款链路完整 + 服务端验票
