# Dreamate Figma PRD（V0.1）

> 基于 2026-01-11 Figma 全量界面截图整理的产品需求，覆盖“世界/云端/恋人/成长/个人”等核心导航与关键流程。若与实际设计稿有出入，以 Figma 为准并在评审中修订。

## 1. 背景与目标
- 用统一的信息架构承接“发现内容 → 进入剧情 → 关系成长 → 付费与复访”的闭环。
- 让用户在 30 秒内完成首次探索与进入对话。
- 用成长、礼物、内容推荐提升留存与付费意愿。

## 2. 用户画像
1. **陪伴型用户**：核心诉求是稳定对话、情绪支持、关系感。
2. **内容探索型用户**：希望通过“世界/云端”找到剧情或场景入口。
3. **成长驱动型用户**：关注任务、进度、成就与成长反馈。

## 3. 信息架构与导航（底部 4 Tab）
| Tab | 目的 | 关键页面 |
| --- | --- | --- |
| 恋人 | 角色关系与对话 | 恋人列表、恋人主页、对话 |
| 探索 | 内容发现与场景库 | 内容流、场景详情、搜索、空镜 |
| 成长 | 任务与成长反馈 | 成长首页、任务列表、内容详情、送礼 |
| 个人 | 个人信息与资产 | 个人主页、资料/设置、最近浏览 |

## 4. 关键用户流程
1. **发现-进入剧情**：探索 → 内容卡片/场景 → 场景详情 → 进入对话。
2. **恋人对话**：恋人列表 → 选择恋人 → 对话 → 关系提升提示。
3. **成长闭环**：成长首页 → 完成任务 → 进度反馈 → 奖励或送礼。
4. **个人管理**：个人主页 → 资产/设置 → 隐私与通知。

## 5. 功能需求（按模块）

### 5.1 探索（内容与场景）
- **FR-001**：探索首页展示内容卡片流，含封面图、标题、简述、热度/标签。
- **FR-002**：点击卡片进入内容详情（可直接开始剧情或进入场景详情）。
- **FR-003**：支持下拉刷新与分页加载；列表加载状态清晰可见。
- **FR-004**：场景列表以“场景卡片”呈现，突出大图与简短引导文案。
- **FR-005**：场景详情页包含：场景介绍、角色推荐、开始按钮、标签。
- **FR-006**：提供搜索入口，支持关键词检索与标签筛选。
- **FR-007**：搜索结果以双列卡片展示，点击进入场景详情。
- **FR-008**：空镜页显示“暂无内容/无结果”与引导操作（返回或推荐内容）。

### 5.2 恋人（角色与对话）
- **FR-009**：恋人列表展示头像、名称、状态/心情与最后一条对话摘要。
- **FR-010**：恋人主页展示角色大图、关系状态、快捷入口（对话/送礼）。
- **FR-011**：对话页支持文本输入、系统消息、卡片消息；底部输入区固定。
- **FR-012**：单个对话页支持“快捷回复/提示卡”入口（位置与样式与 Figma 对齐）。

### 5.3 成长（任务与进度）
- **FR-013**：成长首页展示进度摘要（如等级/亲密度/完成率）与任务入口。
- **FR-014**：任务列表显示状态（未完成/已完成）、奖励、截止信息。
- **FR-015**：成长内容详情页支持长文本阅读与图文混排。

### 5.4 送礼（道具与礼物）
- **FR-016**：送礼弹层提供礼物选择、预览、价格与“确认送礼”。
- **FR-017**：送礼后展示反馈（关系提升/提示文案），并记录在对话中。

### 5.5 个人（我的/设置）
- **FR-018**：个人主页展示头像、昵称、会员状态、资产概览。
- **FR-019**：个人资料页支持编辑基础资料与偏好设置。
- **FR-020**：最近浏览/最近互动列表可查看最近内容或对话。
- **FR-021**：设置项包括隐私、通知、语言、数据导出等。

### 5.6 底部导航
- **FR-022**：底部导航固定 4 项，支持状态保持与选中态高亮。
- **FR-023**：切换 Tab 保持各自滚动位置与内容状态。

## 6. 交互与状态
- **加载态**：列表与详情页展示 skeleton/占位。
- **空态**：云端搜索与列表无结果时显示空镜页。
- **异常态**：网络失败时提供重试与离线提示。

## 7. 数据与对象（简化）
| 对象 | 关键字段 |
| --- | --- |
| Scene | id, title, cover, tags, intro, recommended_roles |
| Lover | id, name, avatar, mood, relationship_level |
| Message | id, role_id, direction, content, created_at |
| GrowthTask | id, title, reward, status, due_at |
| Gift | id, name, icon, price, effect |
| UserProfile | id, name, avatar, membership, coins |

## 8. 关键指标与埋点
- `explore_card_view` / `explore_card_click`
- `scene_search` / `scene_open`
- `lover_open` / `chat_send`
- `growth_task_complete`
- `gift_send_success`
- `profile_edit` / `settings_open`

## 9. 非功能需求
- **性能**：核心列表 P95 首屏 < 2s；对话页渲染 < 1s。
- **稳定性**：关键入口可用性 ≥ 99.5%。
- **隐私**：对话与画像数据默认本地缓存，上传需明确授权。

## 10. 里程碑（建议）
- **M1**：信息架构与导航落地，世界/云端/恋人基础可用。
- **M2**：成长与送礼闭环，埋点上线。
- **M3**：体验优化（空镜/异常态/性能）与数据分析迭代。
