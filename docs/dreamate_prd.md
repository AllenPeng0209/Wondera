# Dreamate 产品需求文档（PRD）

## 1. 背景与机会
- Lovemo 等“AI 固聊”应用已验证情感陪伴赛道的需求，强调“温柔、懂你小心思”的角色化体验，且在 App Store、腾讯应用宝、小米等渠道获得正向反馈及频繁迭代信号。[^appstore][^tencent][^xiaomi]
- Dreamate 旨在以 Expo 打造跨平台 AI 陪伴体验，保留 Lovemo 的成功元素（角色定制、持续问候、会员记忆等），同时强化离线可用性与隐私：以本地数据库为核心，云端仅用于模型推理与收据校验。

## 2. 产品愿景
> “Dreamate 让每位用户都能随时召唤一位熟悉的 AI 伙伴，在任何设备上继续一段温柔的对话。”

愿景要点：
1. **固聊关系**：长期陪伴、可塑造角色人格与记忆。
2. **离线优先**：对话历史、角色设定、记忆等数据都可离线管理，弱网环境仍可查询/编辑。
3. **多端一致**：基于 Expo 构建，快速覆盖 iOS、Android，并为未来的 Web/桌面扩展保留空间。

## 3. 目标与 KPI
| 指标 | 目标 | 备注 |
| --- | --- | --- |
| D7 留存 | ≥ 35% | 结合 Lovemo 的“日常问候”体验。[^appstore] |
| 日均对话轮数 | ≥ 40 | 期待深度互动；本地记忆确保上下文完整。 |

## 4. 用户画像与痛点
1. **孤独陪伴型**：希望每日有人早安/晚安问候，记录心情；担心隐私泄露，需离线存储。
2. **角色创作者**：喜欢自定义世界观与人设，渴望 AI 角色记住细节并持续成长。
3. **情绪支持型**：需要在情绪波动时得到即时回应，期望语气温柔、懂自己。

痛点：
- 传统陌生人社交存在社交压力。
- 依赖云端的 AI 聊天在弱网场景难以回顾历史。
- 记忆能力不足导致角色“忘记”重要事件。

## 5. 用户旅程
1. **获客**：应用商店投放“理想 AI 伙伴”素材、合作 Mergeek 等产品媒体曝光。[^mergeek]
2. **首次体验**：
   - 选择角色模板或自建角色。
   - 指导完成基础设定（称呼、关系、语气、人格标签）。
   - 引导开启“每日问候”与“离线保留”提示。
3. **日常循环**：
   - 推送或本地提醒触发早安/晚安对话。
   - 角色依据本地记忆回忆事件，提供回忆卡片。
   - 需要更深互动时可解锁会员记忆或购买虚拟币触发特殊剧情。
4. **维护与升级**：
   - 用户可在“记忆管理”里编辑或删除 AI 记忆。
   - 可导出/备份本地数据库，切换设备时通过端到端加密同步。

## 6. 功能需求
### 6.1 核心体验
| 功能 | 描述 | 优先级 |
| --- | --- | --- |
| 角色创建与模板 | 选择官方模板（恋人、挚友、导师等）或自定义姓名、背景、语气。 | P0 |
| 多轮对话 | 文字输入（V1）、语音输入（V1.1）、图片输入（V2）；支持长上下文。 | P0 |
| 本地记忆库 | 将事件摘要、标签存入本地数据库，可分类管理；VIP 解锁更大容量。 | P0 |
| 回忆卡片 | 根据记忆生成卡片 feed，供用户复盘或分享。 | P1 |
| 早晚问候任务 | 允许设置提醒时段，本地通知触发“问候脚本”。 | P0 |
| 情绪日记 | 将对话片段标记为心情记录并添加标签，支持离线浏览。 | P1 |

### 6.2 付费与成长
| 功能 | 描述 |
| --- | --- |
| 虚拟币（Moly） | $0.99 起售，用于解锁特殊剧情、语音合成、独家皮肤。[^appstore] |
| VIP 会员（月卡） | $3.99，包含增强记忆容量、无限角色槽、私密加密备份。[^appstore] |
| 激励任务 | 完成每日对话、分享等任务获得少量虚拟币，提升活跃。 |

### 6.3 合规与隐私
- 首次启动展示《Dreamate 软件许可及服务协议》《隐私政策》，沿用 Lovemo 的合规结构并明确离线存储策略。[^tos][^privacy]
- 未成年人须监护人同意；14 岁以下限制付费功能。[^privacy]
- 提供“数据导出、清空对话、注销账号”入口，承诺 15 个工作日处理。[^privacy]

## 7. 内容与品牌语气
- 主色调：暖粉+渐变（可借鉴 Lovemo 的温柔风格）。
- 语气：柔和、细腻、不过度调侃；避免触发敏感话题。
- 文案原则：强调“懂你的小心思”“固聊陪伴”“安心守护”。

## 8. 数据与分析需求
- 本地记录对话轮次、问候完成率、记忆被引用次数，用于在网络恢复时批量上报。
- 集成神策 / TalkingData 进行分渠道分析，遵守隐私合规并在设置页提供关闭选项。[^sdk]
- 设置关键事件：`role_created`、`greeting_completed`、`memory_saved`、`vip_purchased` 等。

## 10. 依赖与风险
- **模型依赖**：需确定可商用中文对话模型（如华为云大模型或其他私有化方案）。[^privacy]
- **离线复杂度**：本地 DB 与云端同步需冲突解决策略。
- **合规**：需在政策更新时同步更新协议；广告 SDK 必须遵守透明化要求。[^sdk]
